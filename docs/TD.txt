Техническое задание на разработку сервиса тестирования сотрудников.



Спроектировать систему  тестирования сотрудников. В системе необходимо предусмотреть  роли - админ и работник. 

Админ может создавать тесты и работников.

Работник имеет имя и должность.
Работник может проходить тесты и видеть свои ранее пройденные тесты.


Тесты состоят из вопросов и вариантов ответов. Правильных ответов может быть несколько или правильного ответа может не быть вообще. Пока не усложняем но возможно в будущем нужно будет предусмотреть ограничение по времени. 
Тест считается пройденным если сотрудник ответил правильно на 80% ответов.


Админ создает тесты. Пока для всех сотрудников но в будущем нужно будет создавать тесты для определенных должностей.

Админ должен видеть всех работников и результаты по пройденным тестам с процентом правильных ответов. Так же админ может посмотреть все ответы по конкретному тесту любого сотрудника.


В процессе разработки можно использовать любые библиотеки. На фронте использовать React или Vue. Будет плюсом использование Typescript. 

На бэк-эд можно использовать Nest.js.
Для БД можно использовать Oracle.

Проект нужно выложить на GitHub и добавить файл Read.me c описанием запуска проекта.

Если нужна дополнительная информация - её можно придумать самому и дополнить ТЗ. 

Не нужно много внимания уделять интерфейсам - для этого можно взять готовую библиотеку по типу  bootstrap.
